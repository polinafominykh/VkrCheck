# VKRGuard - система автоматической проверки ВКР

## Возможности

* автоматическая загрузка и обработка ВКР в форматах PDF и DOCX;
* очистка и сегментация документа на главы и ключевые разделы;
* формальная проверка структуры документа;
* проверка введения по обязательным смысловым блокам;
* смысловая проверка введения с использованием LLM;
* смысловая проверка соответствия содержания главы её названию;
* формирование итогового отчёта в JSON и PDF;
* пакетная обработка нескольких ВКР;
* минимальный веб-интерфейс с возможностью выбора агентов проверки;

## Логика работы

Система строится как мультиагентный пайплайн, в котором каждый агент отвечает за свой уровень анализа:

### 1. Формальный агент

Проверяет базовые формальные критерии:

* наличие обязательных разделов;
* наличие глав верхнего уровня;
* наличие и объём введения;
* наличие списка источников и количество позиций.

### 2. Агент структуры

Проверяет структуру введения:

* актуальность;
* проблема;
* цель;
* задачи;
* практическая значимость;

### 3. Смысловой агент

Использует LLM-модель для:

* оценки смысловой полноты введения;
* оценки соответствия содержания главы её названию;
* формирования содержательных рекомендаций по доработке.

## Архитектура проекта

Основные модули проекта:

* `ingestion/` — загрузка PDF и DOCX;
* `preprocessing/` — очистка и нормализация текста;
* `segmentation/` — выделение глав и ключевых разделов;
* `agents/` — агенты проверки;
* `criteria/` — YAML-файлы с критериями и промптами;
* `core/` — orchestrator, схемы данных;
* `reporting/` — генерация JSON и PDF-отчётов;
* `ui_web/` — минимальный веб-интерфейс;
* `llm/` — клиент для подключения модели.

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Настройка модели

Если используется OpenAI-compatible endpoint, параметры подключения хранятся в файле .env.

GPT_OSS_HOST=http://<host>:<port>/v1
GPT_OSS_KEY=<api_key>
GPT_OSS_MODEL_NAME=<model_name>

## Основные сценарии запуска

### 1. Проверка одной ВКР

```bash
python main.py
```

### 2. Пакетная обработка нескольких ВКР

```bash
python main_batch.py
```

### 3. Запуск веб-интерфейса

```bash
uvicorn web_api:app --reload
```

После запуска интерфейс доступен по адресу:

```text
http://127.0.0.1:8000
```

## Режимы проверки в веб-интерфейсе

В интерфейсе доступны три режима:

* **Формальная проверка** - запускается только формальный агент;
* **Проверка структуры** - запускается агент структуры;
* **Смысловая проверка** - запускается LLM-агент.

Итоговый отчёт формируется только по выбранным агентам.

## Форматы выходных данных

### JSON-отчёт

Содержит:

* сводную статистику;
* список выделенных глав;
* формальные результаты;
* смысловые результаты;
* общий список замечаний и рекомендаций.

### PDF-отчёт

Содержит:

* итоговую сводку;
* перечень основных разделов;
* таблицу формальных проверок;
* таблицу смысловых проверок;
* подробные рекомендации по доработке.

## Поддерживаемые форматы входных файлов

* `.pdf`
* `.docx`

## Логика пайплайна

1. загрузка документа;
2. извлечение текста;
3. очистка текста;
4. сегментация на главы и разделы;
5. запуск выбранных агентов;
6. объединение результатов в единый отчёт;
7. сохранение результатов в JSON и PDF.
